# Catalog Library

A MoMA-inspired flipbook catalog viewer. Browse ~200 PDF catalogs stored on Google Drive with an Issuu-style 3D page-turn reader.

## Local Development

This is a static site — no build step required.

```bash
# Serve locally (needed for History API routing to work)
npx serve .
# or
python3 -m http.server 8080
```

Open `http://localhost:8080`.

## File Structure

```
catalog-app/
├── index.html            # App shell
├── css/style.css         # All styles (design tokens, library, reader)
├── js/
│   ├── app.js            # SPA router (History API)
│   ├── library.js        # Catalog grid, filtering, search
│   ├── reader.js         # 3D flipbook reader
│   └── pdf-renderer.js   # PDF.js wrapper with canvas pool
├── catalogs.json         # Catalog manifest (edit or auto-generate)
├── covers/               # Cover image JPEGs (generated by script)
├── worker/
│   └── proxy.js          # Cloudflare Worker: Google Drive CORS proxy
├── scripts/
│   ├── build-catalog.js  # Enumerate Drive folder → catalogs.json
│   └── generate-covers.js# Render page 1 of each PDF → covers/*.jpg
└── _redirects            # Cloudflare Pages: SPA catch-all route
```

## Setup

### 1. Build the catalog manifest

Install a Google API key (no OAuth needed for public folders):

```bash
GOOGLE_API_KEY=your_key node scripts/build-catalog.js
```

This writes `catalogs.json` with all PDFs found in the configured Drive folder.

### 2. Generate cover images (optional but recommended)

```bash
npm install canvas pdfjs-dist node-fetch
node scripts/generate-covers.js
```

Cover JPEGs are saved to `covers/`. The script updates `pageCount` in `catalogs.json`.

### 3. Deploy the Cloudflare Worker proxy

PDF.js cannot fetch Google Drive files directly due to CORS. The Worker proxy solves this.

```bash
npm install -g wrangler
wrangler login
wrangler deploy worker/proxy.js --name catalog-proxy
```

Then update `window.APP_CONFIG.workerUrl` in `index.html`:

```html
window.APP_CONFIG = {
  workerUrl: 'https://catalog-proxy.your-account.workers.dev',
  ...
};
```

### 4. Deploy to Cloudflare Pages

```bash
wrangler pages deploy . --project-name catalog-app
```

The `_redirects` file ensures all routes return `index.html` for SPA routing.

## Configuration

Edit the `APP_CONFIG` block in `index.html`:

| Key | Default | Description |
|-----|---------|-------------|
| `workerUrl` | `''` | Cloudflare Worker proxy URL |
| `driveFolderId` | `'1U2_lMcMB8IxVL-1HVIHtF8N9FarJ18aL'` | Google Drive folder |

## Keyboard Shortcuts (Reader)

| Key | Action |
|-----|--------|
| `←` / `→` | Previous / Next page |
| `Home` / `End` | First / Last page |
| `F` | Toggle fullscreen |
| `T` | Toggle thumbnail strip |
| `+` / `-` | Zoom in / out |
| `Esc` | Close reader |
| `?` | Show keyboard shortcuts |

## Tech Stack

- **Vanilla JS** — no framework, no build step
- **PDF.js** (v3.11) — PDF rendering via CDN
- **CSS 3D transforms** — `perspective(2000px)`, `rotateY` for page flip
- **History API** — SPA routing
- **IntersectionObserver** — lazy-load cover images
- **Cloudflare Pages** + **Cloudflare Workers** — hosting + CORS proxy
